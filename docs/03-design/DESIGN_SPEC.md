# DESIGN_SPEC.md — 研經網站設計工藝規格

> 對應：STUDY_PAGE_SPEC.md（版面/互動）、study_page_types.ts（契約）。
> 本檔只談**設計層**：identity、字體、色彩、L1/L2 視覺語言、狀態、無障礙、文案。
> 不含實作程式。資料契約不因本檔改變。

---

## 0. 設計命題（thesis）

**「文字本身就是介面。」** 這不是一個有 hero banner 的行銷網站，是一個讀經的場所。最具代表性的東西不是 logo、不是大數字，而是**同一段教法被兩種語言並列持有、逐行對齊**這件事。所以：

- 頁面打開，主角是經文閱讀面，不是 chrome。
- 唯一的「signature」是**對位**——巴利與漢譯的精確對應，以及需要時才顯露的分析層（DPD）。
- 一切裝飾若不服務「讀得久、讀得準、分得清正典與輔助」，一律刪。

---

## 1. 識別方向：素材取自主題本身

研經工具的素材世界是：**貝葉寫本、根本文與旁註的層次、寫本的克制與安靜。** 我們抽象地用它，不做字面裝飾（不用蓮花、不用金佛、不把僧袍橙當主題色鋪滿）。

工藝的真正難點是**排版**：拉丁化巴利的完整變音符號（ṃ ñ ṅ ṭ ḍ ṇ ḷ ā ī ū）與繁體中文並排，兩者都要好看、且能久讀。這才是這個網站「精」或「不精」的分水嶺。

---

## 2. 為何不是 AI 套版（自我批判）

設計 skill 點名三種 AI 預設長相，本案逐一避開：

| AI 預設 | 本案如何不同 |
|---|---|
| 奶油底 #F4F1EA + 高對比襯線 + 赭紅 accent | 紙色刻意偏赭、明顯異於該奶油值；強調色用**燒灼藏紅**非赭紅，且**只**用於對齊焦點，不當裝飾 |
| 近黑底 + 單一螢光綠/朱紅 | 主題為淺色閱讀面（雙語久讀更友善），深色僅為次要變體 |
| 報紙式細線 + 零圓角 + 密欄 | 採對位閱讀節奏，非新聞密欄；分隔線低調且服務對齊，不為風格而存在 |

**關鍵差異化（也是本案的 signature）**：色盤是**雙色溫系統**——暖色承載「正典」、冷灰承載「AI 待核」。色溫差本身編碼了 L1/L2 的信任層級。套版沒有這個語意層。

## 3. Token 系統

### 3.1 色彩（淺色，主）
以「貝葉上的墨，旁有鉛筆批註」為邏輯。每個值都有語意職責，不可挪用。

```
--paper        #E9E0CC   正典經文閱讀面（骨色偏赭，刻意≠#F4F1EA）
--surface      #F4EFE3   chrome / 卡片面（比 paper 亮一階）
--ink          #2A2622   正典墨色（暖黑帶赭褐，非純黑 #000）
--ink-soft     #6B6358   次級文字、輔助說明
--thread       #B06A1F   燒灼藏紅：唯一焦點/對齊強調色，僅用於 active 綁定與其指示
--margin       #5E6B6E   L2「鉛筆」冷灰（刻意冷於 ink；溫度差＝語意）
--margin-bg    #E6E8E4   L2 區塊極淡冷底
--rule         #D8CFB9   分隔線（paper 上的低調線）
--draft        #9A7B2E   未校稿標記（暗芥黃，與 thread 區隔）
```

**規則：** `--thread` 是全站唯一的鮮色，把「大膽」全押在它一處（對齊焦點）。其餘一律安靜。`--ink`（暖）與 `--margin`（冷）的溫度對立，是 L1/L2 的第一道視覺訊號。

### 3.2 色彩（深色，次要變體「夜誦」）
```
--paper #1A1A1E   --ink #E8E2D4(暖,如燈下)   --thread #D8923A   --margin #8A9498(冷灰)
```
深色僅作可選主題，預設淺色。小字變音符號與中文在深色易疲勞，故不設為預設。

### 3.3 字體（主題決定，非順手）
全部開源、可自架（符合本專案離線/開放授權立場）。

| 角色 | 字體 | 理由 |
|---|---|---|
| 巴利原文（顯示+內文） | **Gentium Plus**（SIL OFL） | 為延伸拉丁/IAST 學術轉寫設計，變音符號是**真正畫出來的**而非合成；巴利排版的學界標準 |
| 漢譯正典 | **思源宋體 TC / Noto Serif CJK TC**（OFL） | 宋體的學術沉靜，與 Gentium 同為襯線、字色協調 |
| UI / 標籤 / chrome | Noto Sans TC + Inter（OFL） | 安靜退場，不與經文爭視覺 |
| 分析資料（DPD 詞形碼、id、token 數據） | **IBM Plex Mono**（OFL） | 等寬＝「這是分析/資料層」的誠實訊號 |

**配對理由**：Gentium 與思源宋體都是襯線、x-height 與字色可調和，雙語並排不打架。等寬只給「資料」，不給「經文」——字體本身就在區分「經文 vs 標註 vs 數據」三種內容。

### 3.4 字級與行高（久讀的關鍵）
變音符號需要行高餘裕，中文需要更鬆的行距。

```
巴利內文    19–20px / line-height 1.7   （變音符號不可被上行裁切）
漢譯內文    18px    / line-height 1.9   （中文久讀需更鬆）
經名顯示    28–32px Gentium
DPD 資料    13–14px IBM Plex Mono
標籤/徽章   12px Noto Sans
量度 measure 巴利欄理想 ~58–66 字元寬；超過則閱讀疲勞
```

## 4. L1 / L2 的材質語言（signature）

不靠「加徽章」，靠材質區分。研經傳統裡：**根本文是定稿的墨，註解是頁邊的鉛筆字、視覺上從屬於根本文。** 我們照搬這個真實層次。

| 內容 | 材質隱喻 | 具體處理 |
|---|---|---|
| 巴利原文、漢譯原文（L1） | 墨、定稿 | `--ink` 暖色、實底、主字級、無徽章、無底色 |
| 漢譯逐字白話、研經卡、概要（L2） | 鉛筆旁註、待核 | `--margin` 冷灰、次字級、`--margin-bg` 淡冷底或左側 2px `--margin` 邊框、小寫「AI」標 |
| 未校稿（draft） | 鉛筆且未擦定 | 加 `--draft` 小點/小標「未校稿」 |

**鐵則：**
- L2 在任何排版下，**色溫更冷、對比更低、字級更小、位置更從屬**。讀者餘光就能分辨「這是經文 / 這是輔助」，不必讀徽章。
- 漢譯欄是混合欄：原文是墨、逐字白話是鉛筆。兩者**不可同字級同色**，白話永遠像批註、不像經文。

## 5. 版面工藝

- **對位節奏**：三欄以 segment 為列對齊（見 STUDY_PAGE_SPEC §2）。列與列之間留白要能讓眼睛「橫向掃過去對照」，不是塞滿。
- **焦點之線（signature 的動作）**：focus 某 segment 時，三欄該列以 `--thread` 極細左緣線 + 極淡暖底相連，視覺上「把同一段教法的三種呈現綁成一束」。這條線編碼的是真實結構（它們是同一段），不是裝飾。
- **欄寬**：巴利 40% / 漢譯 35% / 研經 25%（桌機），巴利欄寬度服從上述 measure 上限。
- **chrome 退場**：頁首、控制項用 `--surface` 與 sans，安靜；經文面用 `--paper` 與襯線，主導。視覺重量永遠在經文。
- **圓角**：克制小圓角（卡片/浮層 ~4px），非零圓角報紙感、亦非大圓角 App 感。

---

## 6. 動態（少即是多）

- 經文閱讀面**無環境動畫、無 scroll-jacking**（會立刻顯得 AI 生成、且干擾閱讀）。
- 僅兩處微互動：
  1. 焦點之線：state 切換 ~120ms ease，opacity + 1px 線淡入。
  2. DPD popover：自被點 token 位置快速 fade+輕微 scale（~100ms）。
- `prefers-reduced-motion` 一律尊重：關閉上述過場，只留即時狀態變化。

---

## 7. 狀態即設計（失敗與空白是方向，不是情緒）

依設計 skill：錯誤不道歉、不含糊；空白是邀請。但本案多數「空」其實是**學術事實**，故語氣為「平靜陳述」而非「錯誤」。

| 情境 | 處理與文案（介面語氣，非個人語氣） |
|---|---|
| 無漢譯平行 `chinese:null` | 欄 B 該列以 `--margin` 冷灰排一行：「此段無對應漢譯平行」，下可附小字「本經多處如此」。**不留空、不道歉、不假造。** |
| DPD 查無詞 `dpd_id:null` | popover：「DPD 尚未收錄此詞」。陳述事實，不致歉。 |
| 逐字白話關閉（預設） | 經文乾淨呈現；一個安靜控制項：「顯示逐字白話」（開→「隱藏逐字白話」）。 |
| 未校稿 draft | 冷灰旁註再加 `--draft`「未校稿」小標。 |
| 載入中 | 經文骨架以 `--rule` 淡線佔位，不用旋轉 spinner（與閱讀場所氣質不符）。 |

## 8. 無障礙與久讀舒適

- **變音符號完整性**：確保字體堆疊第一順位即含巴利變音符號，避免 fallback 字體把 ṃ/ḷ/ṅ 換成豆腐或錯字形。這是巴利站的第一無障礙責任。
- **對比**：`--ink` on `--paper` 須達 WCAG AA 內文標準（暖黑/骨色為高對比，OK）；`--thread`、`--margin` 用於線/狀態/次級文字，不作正文。
- **鍵盤焦點**：token 與 segment 可鍵盤導覽，焦點環清晰可見（用 `--thread`）。
- **觸控目標**：巴利 token 小，行動裝置須加 padding 使可點區 ≥ 32–44px，避免誤觸鄰字。
- **可關閉**：逐字白話、研經欄皆可收，讓只想純讀經文的人得到無干擾版面。

---

## 9. 文案語氣（介面的詞彙就是導引）

- **對使用者不講內部術語。** `L1/L2`、`segment`、`grounding` 是我們的內部詞；使用者只看到「原文 / 逐字白話 / 研經卡 / 概要」。介面命名依「使用者能控制與辨認的東西」，不依系統怎麼蓋。
- **平靜、學術、不行銷。** 無驚嘆號、無「立即」「強大」。動詞用平白主動式：「顯示逐字白話」，不是「啟用圖層」。
- **同一動作全程同名**：控制項叫「顯示逐字白話」，開啟後狀態與說明都用一致詞彙。
- **AI 內容誠實標示但不喧賓奪主**：小寫「AI」+ 冷灰材質已足夠；不需大字警語。

## 10. 本檔與既有規格的關係

- 不改資料契約（study_page_types.ts 不動）。
- STUDY_PAGE_SPEC §3「L1/L2 視覺界線」由本檔 §4 具體化為材質/色溫語言。
- 實作時，先依 STUDY_PAGE_SPEC 把結構與狀態做對，再套本檔的 token 與材質。**結構正確優先於美術**——焦點之線、材質區分都是疊在正確 segment 對齊之上的。
