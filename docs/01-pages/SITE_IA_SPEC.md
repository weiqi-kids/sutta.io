# SITE_IA_SPEC.md — 整站資訊架構規格（D-5 + D-6）

> 把已定的各頁面連成一個站:站點地圖、定址、目錄、首頁、導覽、交叉引用、i18n。
> 收口規格層的「連接組織」。延續 L1/L2 紀律、DESIGN 材質與文案、四級出處。

---

## 0. 範圍與 UI 語言決策

**UI 語言(已定,可否決)**:V1 **繁體中文為主介面**;工程上 **i18n-ready**(UI 字串全外部化,§9);**內容維持多語**(巴利恆巴利、DPD 釋義為英文、中文白話為 L2 橋接)。英文介面延 V2。

**V1 範圍**:中部 MN 一集的瀏覽/閱讀/搜尋/字典 + 此經背景 + about。
**V2**:多集、世界總覽頁、英文 UI、對話/語意搜尋。

---

## 1. 站點地圖

```
/                     首頁 / 進入點(D-6)
/browse               目錄(集→經)
/read/{suttaId}       研經頁;深連結段:/read/{suttaId}#{segmentId}
/search               搜尋頁
/lexicon/{key}        字典頁(DPD 字 或 DPPN 專名,§6)
/world                早期佛典的世界(CONTEXT 總覽,🅥V2)
/about                關於:非官方聲明、來源、署名、授權
```

---

## 2. 定址（canonical addressing）

沿用既有 ID,不另發明:

| 對象 | 識別 | URL |
|---|---|---|
| 經 | `suttaId`（mn1） | `/read/mn1` |
| 段(segment) | `segment_id`（mn1:1.1） | `/read/mn1#mn1:1.1`（fragment,靜態友善） |
| 段落(passage) | `passage_id`（mn1:p1） | 研經頁內錨點 |
| 字 / 專名 | lemma / 專名 key | `/lexicon/{key}` |

- **每個 segment、每個詞條皆可深連結**——研經工具的引用/分享是核心需求,務必支援。
- fragment 進入 → 自動捲動 + 該 segment 高亮(沿用 activeSegmentId)。

---

## 3. 目錄 / 瀏覽（D-5）

- 階層:集(中部) → 經清單(經號 + 巴利名 + 中文名)→ 研經頁。
- 經清單可依經號排序;顯示是否有阿含平行、是否已校稿(進度透明)。
- V1 僅中部;結構預留多集(集為第一層)。

---

## 4. 首頁 / 進入點（D-6）

依 DESIGN:**無行銷式 hero**,進入點即「安靜的入口」。
- 一句定位 + 非官方/獨立工具聲明(連 /about)。
- 主要入口(平鋪,不誇張):瀏覽中部、搜尋、字典、(世界 🅥V2)。
- 可精選一部代表經作為閱讀入口示範。
- 文案平靜、學術、不行銷(DESIGN §9)。

---

## 5. 全站導覽

- 頂部固定:站名 → 首頁、瀏覽、搜尋、字典、關於。
- **麵包屑**:`中部 › 根本法門經`(研經頁);詞條頁:`字典 › {詞}`。
- 研經頁 上/下一經(StudyHeader 的 onPrev/onNext)= 在目錄順序內前後移動。
- 導覽 chrome 用 sans + `--surface`,安靜退場(DESIGN §5)。

---

## 6. 經與經之間的交叉引用

研經頁「相關」區(可摺疊)提供:
- **阿含對照**(已在 passage,L1,段落級)。
- **其他語言平行**(SuttaCentral parallels;標來源)。
- **人地事連結**:經中出現的專名 → 詞條頁(DPPN,四級出處)。
- **此經背景**面板(CONTEXT A)。
- 所有外連標明來源層級,不混 L1/L2/學界。

---

## 7. 深連結與分享

- 任一 segment/詞條有穩定 URL,可複製分享、可作引用定位。
- 瀏覽器前進/後退正確;fragment 改變即捲動+高亮,不整頁重載。
- 可離線(靜態 + fragment)。

---

## 8. 響應式導覽

- 桌機:頂部完整導覽 + 麵包屑。
- 手機:導覽收為選單;研經頁三欄改頁籤(STUDY_PAGE §1);麵包屑簡化為「‹ 中部」。
- 巴利閱讀永遠是手機預設頁籤(沿用既有原則)。

---

## 9. i18n-ready（即使 V1 只出繁中）

- **所有 UI 字串外部化**到字串表(key→文案),不硬寫進元件;預設 `zh-Hant`。
- `<html lang>` 隨 UI 語言;**內容語言獨立於 UI**(巴利段 `lang="pi"`、中文 `lang="zh-Hant"`、英文釋義 `lang="en"`)。
- 各語系字型堆疊正確(DESIGN §3.3:巴利 Gentium、中文思源宋體、英文 Inter、資料 Plex Mono)。
- 內容語言接縫(已知):DPD 釋義為英文;中文白話(L2)為中文橋接,UI 須標明哪塊是哪語。

---

## 10. 驗收

1. 站點地圖所有路由可達;深連結到 segment/詞條正確捲動+高亮。
2. 目錄:集→經→研經頁;經清單顯平行/校稿狀態。
3. 首頁安靜入口 + 非官方聲明,無行銷腔。
4. 麵包屑與上/下一經導覽,順序與目錄一致。
5. 研經頁「相關」區交叉引用,各標來源層級。
6. 前進/後退與 fragment 行為正確,可離線。
7. 手機導覽收合得當,巴利為預設頁籤。
8. UI 字串全外部化(抽查無硬寫);內容各段 `lang` 正確。
