# SEARCH_PAGE_SPEC.md — 搜尋頁規格（D-2）

> 對應 SPEC §4.2、DATA_PIPELINE §6（靜態索引）、DESIGN_SPEC（材質/文案）。
> V1 純 L1、本地靜態索引、零後端。語意搜尋（L2）延 V2。

---

## 0. 範圍

**V1（皆離線、零金鑰、零 runtime 託管 LLM）**：
- L1 靜態索引四種:經文定位、字詞、lemma、全文。
- **語意搜尋(嵌入檢索)**:build 期算嵌入,瀏覽器端小模型即時算查詢向量 + 餘弦比對(§6a)。
- **選配 cross-encoder 重排**:小型重排模型,可離線(§6b)。
- **策展研經問題**:固定查詢於 build 期經 Claude Code/Sonnet 重排並校稿、固化成靜態檔(§6c)。

**V2（需 runtime 託管 LLM + proxy）**:任意自由查詢的**生成式重排**、對話研經。

> 釐清:語意「檢索」≠ 生成。前者(嵌入/cross-encoder)可離線,入 V1;唯有任意查詢的**生成式**重排與對話需線上,留 V2。

## 1. 四種搜尋模式

| 模式 | 輸入 | 比對對象 | 落點 |
|---|---|---|---|
| 經文定位 | 經號/經名（mn1、根本法門經） | sutta 索引 | 跳研經頁該經 |
| 字詞（surface） | 巴利字形（sati、satiṃ） | surface→segment 索引 | 結果列 → 研經頁該 segment |
| lemma（原形） | 字典原形（sati） | lemma 索引（含所有變化形） | 結果列 + 連字典頁 |
| 全文 | 任意字串（巴利或中文） | 全文索引（巴利+漢） | 結果列(高亮命中) |

四種皆查 build 期產生的靜態索引（DATA_PIPELINE §6），runtime 本地比對,即時無延遲。

---

## 2. 巴利變音輸入處理（本頁 UX 關鍵）

使用者常打不出 ā ī ū ṃ ṅ ñ ṭ ḍ ṇ ḷ。必須:
- **變音不敏感比對**:`sati` 命中 `sati`、`satī`;`samadhi` 命中 `samādhi`。索引存正規化(去變音/折疊)鍵 + 原形。
- 支援常見轉寫輸入法(Velthuis 風 `aa ii .m` 等)正規化後比對。
- 結果顯示**完整正確變音**的原形,輸入容錯但呈現嚴謹。
- 中文輸入:繁簡折疊比對(輸入簡體也命中繁體經文),呈現用繁體。

---

## 3. 結果呈現

- 每筆:命中片段(高亮)+ 所屬經/集 + segment 定位 → 點擊進研經頁該 segment。
- 巴利用 Gentium、中文用思源宋體(DESIGN §3.3);命中詞 `--thread` 強調。
- 全部 L1 → 無 AI 徽章。
- 結果排序:經文順序(非相關度黑箱),讓使用者可預期。

## 5. 空 / 無結果狀態（DESIGN §7：空白是邀請）

- 無結果:平靜陳述「找不到符合的經文」+ 提示「試試只輸入字根、或放寬變音」。不道歉、不空畫面。
- 初始(未搜):顯示簡短用法提示 + 可能的入口(隨機/代表經)。

## 6. 效能

- 靜態索引隨站打包,首次載入後本地查詢。
- 大索引(全文)考慮分片/惰載;lemma、surface 索引較小可全載。
- 可離線(與 L1 全靜態一致)。

## 7. 語意搜尋與重排（V1 三層,皆離線）

### 6a. 嵌入檢索（V1 核心,離線）
- build 期把每段算成向量 → 靜態嵌入檔(DATA_PIPELINE 新增步驟)。
- **語意表面**:嵌入算在**中文白話(L2)+ 英文 DPD 釋義**上,而非羅馬化巴利——因巴利低資源、嵌入品質不確定;用高資源語言當語意表面,仍命中正確 segment。
- 查詢期:瀏覽器端小型多語嵌入模型(如 multilingual-e5-small 級,~100MB,懶載+IndexedDB 快取)算查詢向量 → 餘弦比對 → 取前 N 候選。
- 即時、可離線、零金鑰。模型僅在使用者首次開語意搜尋時下載。

### 6b. cross-encoder 重排（V1 選配,離線）
- 對 6a 撈回的前 N 候選,用小型 cross-encoder(查詢×文件相關度)重排,提升精準度。
- 仍為**判別式小模型**,非生成式,可離線。
- 我們僅一集、數千段,6a 多半已足夠;6b 為「不夠精準時才補」的選配層。

### 6c. 策展研經問題（V1,build 期 Sonnet 重排,靜態）
- 每部經預設幾個策展問題(「這部經在問什麼」「找談 X 的段落」)。
- 這些**固定查詢**於 build 期經 **Claude Code / Sonnet** 做生成式重排 → 人工校稿 → **固化成靜態結果檔**(L2 任務 T5)。
- 訪客點策展問題 → 載入預排好、審過的高品質結果。零 runtime、零金鑰、可離線。
- 屬 L2:結果標 AI 徽章 + 依據 segment。

> 為何策展可、任意不可:固定查詢能 build 期算好固化;任意查詢事先不知訪客打什麼,生成式重排只能 runtime(V2)。Claude Code 是 build 期執行器,非公開後端。

---

## 8. L1 / L2 分區

- L1 四種搜尋結果:單一列表、無徽章。
- 語意檢索(6a/6b)結果:可與 L1 並列,但標示「語意」;6a/6b 為檢索非生成,不掛 AI 徽章(無生成內容)。
- 策展問題(6c)結果:**獨立分區** + AI 徽章 + L2 材質(含生成式重排),不與 L1 混排。
- V2 任意生成式重排/對話:同樣獨立分區、AI 徽章。

## 10. 驗收

1. 四種 L1 模式皆可查、落點正確。
2. 無變音輸入(`samadhi`)命中有變音原文(`samādhi`)。
3. 簡體輸入命中繁體經文。
4. 結果點擊正確跳研經頁對應 segment。
5. 無結果顯示引導文案,非空白/非道歉。
6. **V1 全程零網路、零託管 LLM**(含語意搜尋:模型本地、嵌入靜態)。
7. 語意搜尋:中文/英文查詢能命中對應 segment(語意表面為白話+釋義);模型懶載。
8. 策展問題:點擊載入預排+審過的靜態結果,標 AI 徽章。
9. L1 結果無徽章;6c/V2 生成內容獨立分區+AI 徽章,不與 L1 混排。
