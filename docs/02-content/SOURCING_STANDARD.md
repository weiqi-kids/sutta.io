# SOURCING_STANDARD.md — 研究級來源標準（J-7）

> 適用：背景脈絡層(CONTEXT)的「學界」與「註釋」內容,以及任何非經文的歷史陳述。
> 不適用：經文本身(那是 L1 正典,以 segment 定位即可)。
> 與四級出處(CONTEXT §3 / type `Provenance`)對接:每條事實落為 `SourcedFact`。

---

## 0. 一句原則

**這層的每一句話都要能回答「誰說的、出自哪、有沒有定論」。** 答不出就不寫,或標為待補。AI 只能整理有來源的說法,不能自己生出史實。

---

## 1. 可引用來源的層級

| Provenance | 可引用什麼 | 必附 |
|---|---|---|
| `canonical` 正典 | 經文本身 | segment_id |
| `commentarial` 註釋 | 具名註釋(如覺音《清淨道論》、各部義疏) | 註釋名 + 出處 |
| `scholarly` 學界 | 同行評審論文、標準學術專著、權威工具書 | 作者 + 書/篇名 + 年 |
| `ai` | 上述之**整理/串接** | grounded_on(所據條目) |

**失格來源(不可作 scholarly 依據)**:匿名網站、無出處的通俗站、把宗派護教當史實者、AI 生成內容當引用、二手轉述而查不到原始者。

---

## 2. 引用規則

- 每條 `scholarly` 事實 = 作者 + 出處 + 年;無此三者不得標 scholarly。
- **絕對年代永不以單一值斷言**:用「傳統繫於 / 學界多繫於 / 無定論」+ 範圍。
- 同一事實有 ≥2 學術立場 → 進「爭議標注」(§3),不擇一當定論。
- 不可把「廣為流傳」當「已成定論」(§5A 即反例)。

---

## 3. 爭議標注規範

當學界無共識:
1. 陳述「此點學界有不同說法」。
2. 列各立場 + 其代表學者/著作(具名)。
3. **不裁決**何者正確;若有「多數採用」可註明,但不寫成定論。
4. UI:`scholarly` 視覺 + 「有不同說法」標 + 可展開各立場。
5. 落型別:一個事實 → 多個 `SourcedFact`(各立場各一條)+ `note` 標爭議。

---

## 4. 年代不確定性(本層最易出錯,單獨立規)

- 古代事件**一律不給單一確定年**。
- 公式:`傳統繫於 X｜學界多繫於 Y｜無定論`,且標出處。
- 經內相對時序(如「此事在某結集後」)可用,但不換算成絕對年。
- AI 不得「為了通順」補上未見於來源的年代。

---

## 5. 種子錨點（真實範例,示範本標準怎麼用）

### 5A. 佛陀年代 —— 高爭議,**示範「不裁決」**
學界長期有多套並存、且明確無共識:
- **修正長部年代(Corrected Long Chronology)**:約 567–487 B.C.E.;Warder(1970)後一度被當近乎確定(約 480 B.C.E. 入滅)。
- **印度短部年代(Short Chronology)**:約 448–368 B.C.E.。
- **現代教科書近似**:約 563–483 B.C.E.(常見,但仍是近似)。
- **Bechert 的重估**:Bechert 主編《When Did the Buddha Live?》(1995)主張修正長部年代「已不可維持」,推動往較晚年代重新檢視。
- **近年再挑戰**:有以天文/考古方法提出新終點者,對上述各說再提異議。

**站上呈現方式**:設「年代問題」說明,列上述各立場(具名)、全部標 `scholarly` + 爭議,不選邊;絕不在經文旁寫死某年。共識僅止於「其確實存在,但確切年代無定論」。

### 5B. 第二次城市化與十六大國 —— **廣泛接受但絕對年代仍辯**
- 「十六大國(Mahājanapadas)」之名單**見於增支部**(屬 `canonical`,可定位經段)。
- 恆河平原「第二次城市化」、城鎮/貨幣/商人階層興起、沙門思潮並起——為廣泛接受的史學/考古框架;但**絕對年代仍有辯論**。

**站上呈現方式**:作時代背景通論,大筆觸描述;凡屬經內者標 `canonical`(連經段),屬學界重建者標 `scholarly` 並把絕對年代寫成範圍/「約」。

---

## 6. AI 在本層的界線

- AI 可:摘要、串接、對照**已具來源**的各立場。
- AI 不可:引入任何未見於來源的年代、事件、因果。
- **反查**(同 L2_GENERATION §5):AI 輸出的每個史實主張,須可對回某條 `SourcedFact`;對不上 → 退回待補,不上線。

---

## 7. 與型別對接

每條事實 → `SourcedFact { provenance, content, source_ref?, note?, review_status? }`(study_page_types.ts §6)。
- `provenance` 決定視覺(design-tokens `--prov-*`)。
- 爭議 → 多條 `SourcedFact` + `note`。
- `ai` 條目額外帶 `review_status`,校稿後始上線。
