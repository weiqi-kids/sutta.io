# 三藏原文解析 — AI 輔助巴利研經工具

> 工作代號:TIPITAKA-LENS
> 一句話:以**巴利語三藏的逐字語法解析**為骨幹,並排**漢譯阿含對照**,再用 LLM 在「有真實資料可依據」的前提下,生成漢譯白話、概要與研經輔助——而且嚴守「正典 / AI」的信任界線。

---

## 這是什麼

一個給認真讀經的人用的研經網站。打開一部經,你會看到:

- **巴利原文**,每個字可點,查得到字根、詞形、複合詞拆解(資料來自 Digital Pāḷi Dictionary)。
- **漢譯白話**,逐字對照,是 AI 依據巴利的真實語法生成的現代中文解釋。
- **漢譯阿含對照**,另一個古代傳本(CBETA),段落級並排。
- **研經卡、章節概要、此經背景**(當時的人、地、時代、在典的天災人禍)。
- 全文 / 字詞 / 原形 / **語意**搜尋,以及字典頁。

對標的是「聖經原文研經工具」那類網站,但巴利/漢譯的語言關係更複雜,也更有研究價值。

---

## 核心理念:把 AI 放在它能加值、又不會說謊的位置

佛教是教義敏感的領域,AI 一旦亂講,可信度就毀了。所以整套架構建立在一條界線上:

- **L1 權威層**(經文、字根、詞形、對照)——一律來自權威來源,可追溯,**永不由 AI 生成或改寫**。
- **L2 生成層**(白話、概要、研經卡)——AI 生成,但**每一次都必須以 L1 真實資料為依據**(RAG),且上線前人工校稿。
- 視覺上用「**墨 vs 鉛筆旁註**」區分:正典是定稿的墨(暖色),AI 是頁邊待核的鉛筆(冷灰),連色溫都帶意義。
- 背景/歷史另設**四級出處**(正典 / 註釋 / 學界 / AI),每條事實都能回答「憑什麼這麼說」;絕對年代不寫死、爭議並陳不裁決。

一句判準:**事實走 L1,解釋走 L2,解釋永遠掛在事實上。**

---

## 三個讓它不一樣的地方

1. **漢譯逐字解析其實做得到**——靠的不是讓 AI 編漢字字源(死路),而是利用「漢譯阿含已對齊巴利、巴利又有 DPD 真實字根」,讓 AI 拿真實語法去解釋,有錨、非捏造。
2. **V1 完全靜態、可離線、零金鑰**——所有 AI 內容 build 期預生成 + 校稿後固化;連語意搜尋都用 build 期嵌入 + 瀏覽器端小模型,不需後端、不需 API 金鑰。
3. **研究級出處紀律**——背景與歷史一律具名引用、標明無定論之處,不讓網站變成某一宗派詮釋的傳聲筒。

## 架構一覽

```
資料源(SuttaCentral 巴利 / DPD 詞形 / CBETA 阿含 / DPPN 專名)
   ↓ build 期管線(擷取→DPD join→消歧→對照→白話→索引→嵌入)
靜態預建檔(每部經一個 JSON + 索引 + 嵌入)
   ↓
靜態前端(GitHub Pages)— L1 讀靜態、零 LLM;L2 預生成已固化;語意搜尋本地跑
```

runtime 不接託管 LLM(V1)。對話研經、任意查詢的生成式重排需 runtime 強模型,留 V2。
